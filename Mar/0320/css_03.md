# CSS 박스 모델

## 박스 모델 : 가로와 세로
- 콘텐츠가 들어가는 영역의 크기를 정함
- width;, height;를 통해 차지하는 영역을 정해줌

## 테두리와 모깎기
- 테두리는 웹페이지를 보기좋게 하는데 중요한 역할을 하고 각 요소를 구별하는데 도움이 됨
- 테두리 두께, 테두리 색깔, 테두리 스타일이 대표적인 속성
- 테두리에는 보통 픽셀 단위를 쓰는데 테두리 두께는 일반적으로 얇고 웬만해선 규격이 변하지 않기 때문
- box-sizing: border-box 같은 속성을 주면 border 두께만큼 영역의 안쪽을 차지하기 때문에 콘텐츠 영역의 절대적인 넓이가 줄어듬
- border-style: dashed groove none dotted; 처럼 최대 4개의 스타일을 줄 수 있는데 각각 "상우하좌"의 순서
- border-left-color 처럼 한 쪽 면만 스타일을 따로 주는 것도 가능함
- border: 4px solid green; 속기법을 사용하여 한번에 스타일 적용도 가능 (width, style, color 순서가 일반적)
- border-radius 속성은 테두리를 둥글게 할 수 있음. px로 줄 수 있지만 %로도 적용 가능함 (50%를 주면 원형으로 만들 수 있음)

## 패딩
- 패딩이란 콘텐츠와 테두리 사이의 공간을 의미한다. 
- 네 개의 개별 특성을 동시에 적용하는 속기법이 존재
- 값이 하나일때는 전체를, 두개일 때는 상하, 좌우, 세 개일때는 상단, 좌우, 하단, 네 개일때는 각각 상우하좌 순서(시게방향)

## 여백 (Margin)
- 패딩이 안쪽 공간을 차지한다면, 마진은 반대로 테두리의 바깥 공간을 차지함.
- 패딩의 속기법과 동일한 방법으로 적용됨

## 디스플레이 속성
- 인라인 요소는 다른요소들과 같은 줄에 배치되고 블록 요소는 밀어냄
- Inline, Block, Inline-block으로 나뉨
- 요소가 인라인이라면 모든 width는 무시됨
- 반면에 패딩이라면 적용되기는 하지만 나머지 요소에 영향을 주지는 않고 여전히 한 줄에 존재함 (덮여있거나 아래에 깔려있는 느낌)
- 블록 요소는 width, height, padding, margin 등의 스타일이 모든 주위 요소에 영향을 주면서 적용됨(밖으로 밀어냄)
- 인라인 블록은 위의 두가지 요소가 합쳐졌다고 볼 수 있음. 즉, 한 줄에 존재하지만 height, width 등은 블록처럼 적용됨
- none은 사라지는 것이 아니라 단지 보이지 않는 것일 뿐임. 따라서 이를 이용해서 여러가지 동적 효과를 구현 가능함

## CSS 단위 (상대적 단위)
- Absolute 유닛에는 px, pt, cm, in, mm 등이 있으나 실질적으로 쓰이는 것은 px 뿐임
- 반면에 Relative(em, rem, vh, vw, % 등) 단위는 꽤 빈번하게 사용됨
- '%'는 기본적으로 부모 값을 기준으로 하는 상대적 값임 (높이나 너비의 픽셀을 기준으로 하는 것이 보편적)

### EM
- EM은 REM처럼 특성에 따라 다른 값이 바뀜
- 예를 들어 폰트의 크기가 1 EM이라면, 부모 속성과 동일한 값을 가지고 2EM 이라면 부모속성의 두배 크기
- 쉽게 이해하자면 부모 속성의 배수라고 보면 됨
- 버튼의 테두리를 둥글게 만들 때, 픽셀로 설정하면 크기가 바뀔 때마다 모서리의 둥근 정도도 같이 바뀌지만 em으로 설정하면 비율을 유지할 수 있음

### REM
- EM이 가진 문제는 단계별로 누적된다는 것 (즉, li같은 여러번 중첩되는 코드를 사용할 경우 급격하게 커지거나 작아질 수 있음)
- REM(Root EM)은 부모 속성에 따라 바뀌지 않고 루트 HTML 요소의 글씨 크기에 따라 바꾼다는 것이 다름
- 즉 문서 전체에서 하나의 폰트를 기준으로 크기를 조절한다는 점
- 스노우볼링을 막아주고 하나의 값을 기준으로 변함
- EM과 REM을 동시에 사용하는 경우도 있는데, 예를들어 버튼에 모깎기를 적용할 때 실제 폰트 크기가 계산되고 나서 거기에 맞춰야 되는 경우 등
- 모깎기를 루트 요소에 따라 달라지게 설정하면 요소이ㅡ 실제 크기를 반영하지 않게 되기 때문
